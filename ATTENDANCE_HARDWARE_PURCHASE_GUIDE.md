## 7. Integration with School Portal

### API Integration Requirements:

**For Biometric Devices:**
- HTTP/REST API support
- JSON data format
- Real-time webhook callbacks
- Cloud synchronization

**For QR Scanners:**
- Camera API access
- Barcode decoding library
- Web browser compatibility
- Offline capability

**Sample Integration Code:**
```typescript
// Biometric device webhook endpoint
app.post('/api/attendance/biometric-checkin', async (req, res) => {
  const { deviceId, userId, timestamp, verificationType } = req.body;

  // Verify device is registered
  const device = await getDevice(deviceId);
  if (!device) return res.status(401).json({ error: 'Invalid device' });

  // Record attendance
  await recordAttendance({
    studentId: userId,
    date: new Date(timestamp),
    status: 'present',
    checkInMethod: 'biometric',
    deviceId,
  });

  res.json({ success: true, message: 'Check-in recorded' });
});
```

## 14. Frequently Asked Questions

**Q: Can biometric data be stolen?**
A: Modern devices store encrypted templates, not actual fingerprints. Templates cannot be reverse-engineered.

**Q: What happens during power outages?**
A: Battery backup ensures 4-8 hours of operation. Cloud sync resumes after power returns.

**Q: Do students need smartphones for QR codes?**
A: No. Schools provide printed QR cards. Students just scan at the gate.

**Q: How long does facial recognition enrollment take?**
A: 10-20 seconds per student. A school of 500 students can be enrolled in 2-3 days.

**Q: Can the system work offline?**
A: Yes. Devices store data locally and sync when internet is available.

**Q: What about students who forget their QR cards?**
A: Backup fingerprint/manual check-in options available.
